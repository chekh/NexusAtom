# Решение ENG-002: Нормализация Slug (имени файла)

## Проблема
Как формировать читаемую часть имени файла (`slug`) после ID (`ENG-001-...`)?
Нужен баланс между удобством для человека (русский язык, полные слова) и надёжностью для системы (скрипты, URL, кросс-платформенность).

## Решение
Принят стандарт **Strict ASCII Kebab-Case**.

### 1. Правила формирования Slug
| Правило | Пример | Почему |
|---------|--------|--------|
| **Только ASCII (Latin)** | `vault-store-separation` | Избегаем проблем с кодировками, URL, CLI-скриптами на разных ОС. |
| **Нижний регистр** | `vault` не `Vault` | Unix-системы чувствительны к регистру. Единый стиль упрощает поиск. |
| **Kebab-case** | `vault-store` не `vault_store` | Стандарт для веб-ссылок, лучше читаемость. |
| **Транслитерация** | `интеграция` → `integratsiya` | Сохраняем фонетику для поиска. Используем простую таблицу (GOST/RFC). |
| **Удаление стоп-слов** | "Как сделать..." → `how-to` | Сокращаем длину. Оставляем только смысловые ядра. |
| **Лимит длины** | Макс. 50 символов | Длинные имена ломают вывод в терминале и усложняют навигацию. |

### 2. Алгоритм генерации (псевдокод)
```python
def generate_slug(title: str, resource_code: str) -> str:
    # 1. Нижний регистр
    slug = title.lower()
    # 2. Транслитерация (кириллица → латиница)
    slug = transliterate(slug)  # "интеграция" → "integratsiya"
    # 3. Замена не-букв на дефис
    slug = re.sub(r'[^a-z0-9]+', '-', slug)
    # 4. Удаление стоп-слов (опционально)
    slug = remove_stopwords(slug)  # "the", "a", "kak", "sdelat"
    # 5. Очистка краёв и сжатие дефисов
    slug = re.sub(r'-+', '-', slug).strip('-')
    # 6. Обрезка по длине
    return slug[:50]
```

### 3. Примеры
| Заголовок (человек) | Сгенерированный Slug | Итоговое имя файла |
|---------------------|----------------------|-------------------|
| "Разделение Vault и Store" | `vault-store-separation` | `ENG-001-vault-store-separation.md` |
| "Как настроить MCP сервер" | `setup-mcp-server` | `ENG-003-setup-mcp-server.md` |
| "Интеграция с Jira: гайд" | `integratsiya-jira-guide` | `ENG-004-integratsiya-jira-guide.md` |
| "Bug: memory leak in parser" | `bug-memory-leak-parser` | `ENG-005-bug-memory-leak-parser.md` |

## Преимущества
1.  **Надёжность**: Файлы открываются на любой ОС (Linux, macOS, Windows, WSL).
2.  **URL-friendly**: Можно использовать ID+slug как часть веб-адреса без экранирования.
3.  **Поиск**: `grep "ENG-015"` находит файл, даже если пользователь забыл точный заголовок.
4.  **Эргономика**: Короткие, предсказуемые имена не "разъезжаются" в интерфейсе.

## Исключения
- Для ресурсов с не-техническим контентом (например, `NB` - Notebook) допускается **локальное переопределение** в `_contract/registry.yaml`: `allow_unicode_slugs: true`. Но по умолчанию — строго ASCII.
