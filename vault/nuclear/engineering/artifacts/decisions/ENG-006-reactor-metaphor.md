# ADR: Использование метафоры «Реактор» для брендинга и навигации

## Контекст
Для улучшения запоминаемости и ментальной модели системы предложена аналогия с ядерным реактором.

## Карта соответствий
| Термин | Компонент NexusAtom | Функция в системе |
|--------|---------------------|-------------------|
| `Atom` | Узел данных / контейнер SSOT | Единица знания или доставки (`spec`, `note`, `adr`, `task`) |
| `Reactor` | Ядро оркестратора + индексатор | Среда, где атомы валидируются, связываются и запускаются в работу |
| `Critical Mass` | Порог готовности к доставке | Состояние, когда атом/эпик набрал аппрувы, валидный `manifest` и пройденные тесты → готов к merge/release |
| `Chain Reaction` | Граф зависимостей + impact propagation | Автообновление downstream-атомов, инвалидация кэша, запуск CI-цепочек при изменении upstream |
| `Control Rods` | FSM-гейты, Pydantic strict, CI-чеки | Механизмы сдерживания: блокируют «разгон» до прохождения валидации |
| `Moderator` | Human-in-the-loop / ревью | Регулирует скорость перехода статусов, принимает решения на границах |
| `Coolant` | Логирование, трассировка, архивация | Отводит «тепло» (шум, устаревший контекст) в `archive/` без потери истории |
| `Spent Fuel` | `archived` / `tech_debt` | Безопасно сохранённые, но неактивные атомы |

## Решение
✅ Использовать метафору **только в документации, дашбордах и внутреннем жаргоне**. В CLI/API оставлять технические имена (`nexus atom create`, `nexus validate`).

## Обоснование
1. **Устойчивый ментальный образ**: Команда и агенты быстрее усваивают архитектуру через аналогию.
2. **Естественная терминология для метрик**: `Reaction Rate`, `Mass Index`, `Neutron Flux`, `Core Temp`.
3. **Чёткое разделение**: `Atom` (единица) → `Reactor` (движок) → `Critical Mass` (гейт) → `Chain Reaction` (автопропагация).
4. **Безопасность**: Акцент на **управляемой энергии** (реактор IV поколения, точный контроль). Слоган: `Controlled energy for complex delivery.`

## Риски и митигация
| Риск | Решение |
|------|---------|
| Ассоциация с разрушением/авариями | Акцент в брендинге на управляемой энергии, позитивный слоган |
| Перегрузка терминологией в CLI | В CLI оставлять технические имена. Аналоги → только в дашбордах, документации |
| Размытие технических сущностей | Зафиксировать глоссарий в `docs/glossary.md` с прямой маппингой на код |
