# Решение ENG-001: Система коротких идентификаторов

## Проблема
Предыдущие варианты именования (`ENG-ADR-015`, `MET.ANALYSIS...`) были слишком длинными для ввода в промптах и сложными для поддержки единого реестра.

## Решение
Вводится стандарт **Resource-Sequence**.

### 1. Формат ID
`{RESOURCE}-{SEQ}`
- **RESOURCE**: Код ресурса (например, `ENG`).
- **SEQ**: Сквозная нумерация (например, `001`).
- **Пример**: `ENG-015`.

### 2. Именование файлов
`{ID}-{slug}.md`
- Пример: `ENG-015-vault-store-separation.md`.
- `slug` — краткое описание на английском (kebab-case).

### 3. Реестр
В `_contract/registry.yaml` хранится **единый счетчик** `last_id` для всего ресурса, независимо от типа документа.

## Преимущества
- **Эргономика**: Легко диктовать агенту ("Открой ENG-15").
- **Простота**: Один инкрементальный счетчик вместо матрицы типов.
- **Уникальность**: Гарантирована в рамках Ресурса.
