# ADR: Логическая архивация (статус) вместо физического перемещения

## Контекст
В процессе обсуждения возник вопрос: как архивировать атомы? Варианты:
1. **Физическое перемещение**: `git mv atom/ → _archive/`
2. **Логическая архивация**: Меняется только `status: archived` в `_meta.yaml`, файл остаётся на месте

## Проблемы физического перемещения
- **Ломает ссылки**: Относительные пути перестают работать, требуют редиректов или ручного обновления.
- **Скрывает историю**: Архивированные ADR/Specs исчезают из вида, теряется контекст для обобщающих документов.
- **Усложняет мердж**: Обобщающие документы (`architecture/overview.md`) требуют ручного обновления при каждом перемещении.
- **Риск потери**: `git mv` сохраняет историю, но усложняет `blame` и трассировку.

## Решение
✅ **Логическая архивация по умолчанию**:
- Файлы **никогда не перемещаются**. Меняется только `status` в `_meta.yaml`.
- Индексатор фильтрует по `status` при поиске.
- Ссылки по `id` остаются рабочими независимо от статуса.
- Обобщающие документы рендерятся динамически из графа (`{{query: ...}}`).

**Опционально**: Физическая очистка через `nexus vault cleanup --physical` для энтерпрайз-сценариев (комплаенс, производительность). Требует валидации манифестов и явного подтверждения.

## Обоснование
1. **Целостность ссылок**: `[[id]]` резолвится графом, не зависит от физического пути.
2. **История сохраняется**: Архивированные узлы остаются в графе для аудита и онбординга.
3. **Динамическая агрегация**: Обзорные документы всегда актуальны, не требуют ручного мерджа.
4. **Гибкость**: 95% случаев покрываются логическим архивом. Физическая очистка — опция для масштабных проектов.

## Последствия
- [ ] `_meta.yaml` расширен полями `status`, `archived_at`, `archive_reason`, `superseded_by`
- [ ] Индексатор фильтрует по `status` по умолчанию, `--include-archived` для полного поиска
- [ ] `nexus atom archive` меняет статус, не двигает файлы
- [ ] `nexus vault cleanup --physical` — опциональная команда для физической архивации

## Статус
✅ Принято. Реализовано в `principle/logical_archiving.md`.
