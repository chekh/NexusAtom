# Требование: Навигация по ID, не по путям

## Описание
Все ссылки между артефактами используют стабильный `id`, не относительные файловые пути. Граф-резолвер подставляет актуальный путь и статус при рендеринге.

## Формат ссылок
```markdown
# В контенте артефакта
См. решение: [[ENG-041]]
Зависит от: [[ENG-041]]
Порождено: [[ENG-018]]
```

## Механика резолвинга
1. Индексатор читает `_meta.yaml` всех файлов → строит маппинг `id → {path, status, type}`.
2. При парсинге контента (`[[id]]`) резолвер:
   - Находит узел по `id`
   - Подставляет путь для навигации
   - Добавляет бейдж статуса (`active`, `deprecated`, `archived`)
   - Валидирует битые ссылки (если `id` не найден)

## Преимущества
- **Физическая стабильность**: Файлы никогда не перемещаются. Меняется только `status`.
- **Целостность при архивации**: Ссылка на `[[ENG-041]]` остаётся рабочей, даже если узел `archived`.
- **Динамическая агрегация**: Обзорные документы рендерятся из графа, не требуют ручного мерджа.

## Критерии приемки
- [ ] `nexus resolve [[id]]` возвращает `{path, status, type, preview}`
- [ ] CI-хук проверяет битые ссылки при коммите
- [ ] При `status: deprecated` резолвер показывает `superseded_by` ссылку
- [ ] `{{query: ...}}` шаблоны в обзорах рендерятся в актуальный контент

## Зависимости
- `meta/registry.yaml` (определение типов и профилей)
- `principle/logical_archiving` (статусы + `supersedes_by`)
- Индексатор графа (NetworkX/Neo4j)
