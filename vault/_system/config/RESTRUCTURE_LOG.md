# Лог реструктуризации `vault/` — v2 (консолидация системных ресурсов)

## Цель
Сгруппировать все системные ресурсы под `_system/` для улучшения навигации и чёткого разделения ответственности.

## Изменения (в дополнение к предыдущему коммиту)

### 1. Консолидация `_system/`
Все системные ресурсы теперь находятся внутри `_system/`:
```
vault/
├── _system/              # 🔒 ВСЯ ИНФРАСТРУКТУРА (всегда вверху)
│   ├── config/           # manifest.yaml, schema.yaml, nuclear_registry.yaml, security_policy.yaml
│   ├── context/          # principles/, workflows/, memory/, history/ (знания и процессы)
│   ├── archive/          # global/ (compliance, retention)
│   ├── meta/             # audit/, indexes/, migrations/ (операционные метаданные)
│   └── shared/           # guides/, templates/, ontologies (кросс-доменные знания)
├── workspace/            # ⚡ Активная работа (inbox, drafts, current)
└── nuclear/              # 📦 Ядерные ресурсы (доставка)
    └── engineering/      # Артефакты разработки
```

### 2. Перемещённые ресурсы
| Откуда | Куда | Примечание |
|--------|------|------------|
| `_system/*.yaml` | `_system/config/*.yaml` | Конфигурационные файлы |
| `context/` | `_system/context/` | Принципы, процессы, память агентов |
| `archive/` | `_system/archive/` | Глобальный архив для комплаенса |
| `meta/` | `_system/meta/` | Операционные метаданные ядра |
| `shared/` | `_system/shared/` | Кросс-доменные гайды и шаблоны |

### 3. Что осталось на корневом уровне
- `workspace/` — поток активной работы (не системный ресурс, а рабочая область)
- `nuclear/` — ядерные ресурсы доставки (ценность проекта, не инфраструктура)

## Почему это лучше
✅ **Навигация**: 1 клик → вся инфраструктура. Не нужно искать `context/`, `archive/`, `meta/` отдельно.  
✅ **Масштабируемость**: Новые системные модули добавляются внутрь `_system/`, корень не раздувается.  
✅ **Ментальная модель**: Чёткое разделение:  
- `_system/` = **как работает** `vault/` (правила, знания, настройки)  
- `workspace/` = **где мы работаем** сейчас (поток)  
- `nuclear/` = **что мы производим** (результат, ценность)  

## Сохранённость данных
✅ Все файлы сохранены, ни один не удалён.  
✅ Все `id` в фронтматтере сохранены (связи `[[id]]` не разорваны).
