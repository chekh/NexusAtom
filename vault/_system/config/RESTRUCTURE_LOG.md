### 2. Фрактальный стандарт ядерного ресурса
Каждый ресурс в `nuclear/` теперь следует единому стандарту:
```
nuclear/{resource_id}/
├── _system/              # 🔒 ОБЯЗАТЕЛЬНО: локальный контракт + зависимости
│   ├── _contract.yaml    # Типы, права, Зоны тьмы, внутренняя структура
│   └── dependencies.yaml # Зависимости от других ядерных ресурсов
├── _archive/             # 🔒 ОБЯЗАТЕЛЬНО: локальный архив для мобильности
├── <custom_structure>/   # ✅ ОПЦИОНАЛЬНО: определяется контрактом
│   ├── workspace/        # Пример: активная работа (inbox, drafts, current)
│   └── artifacts/        # Пример: утверждённая ценность (analysis, decisions...)
```

**Ключевое**: Только `_system/` и `_archive/` — обязательные части стандарта. Всё остальное (`workspace/`, `artifacts/`, `research/`, `docs/`...) определяется в `_contract.yaml` и может быть разным для разных ресурсов.

### 3. Применено к `nuclear/engineering/`
- `_contract.yaml` и `dependencies.yaml` перемещены в `_system/`.
- `workspace/` и `artifacts/` оставлены как пример внутренней структуры (опционально).
- Контракт обновлён с описанием `internal_structure`.

### 4. Итоговая структура `vault/`
```
vault/
├── _system/              # ГЛОБАЛЬНЫЙ МОЗГ (законы физики)
│   └── config/           # manifest.yaml, schema.yaml, nuclear_registry.yaml, security_policy.yaml
│   └── context/          # Принципы, процессы, память агентов, история
│   └── archive/          # Глобальный архив (комплаенс)
│   └── meta/             # Операционные метаданные ядра
│   └── shared/           # Кросс-доменные гайды
│
└── nuclear/              # МИР ЯДЕРНЫХ РЕСУРСОВ (планеты)
    └── engineering/      # <--- ФРАКТАЛЬНЫЙ СТАНДАРТ
        ├── _system/      # Локальный контракт + зависимости
        ├── _archive/     # Локальный архив
        ├── workspace/    # Активная работа (опционально)
        └── artifacts/    # Готовая ценность (опционально)
```

## Преимущества фрактальной модели
✅ **Мобильность**: Скопировать `nuclear/engineering/` в другой проект = готовый модуль с контрактом и историей.  
✅ **Стандартизация**: `_system/` и `_archive/` всегда на одном месте — инструменты знают, где искать контракт.  
✅ **Гибкость**: Внутренняя структура (`workspace/`, `artifacts/`) определяется контрактом, не навязывается ядром.  
✅ **Масштабируемость**: Новые ядерные ресурсы добавляются в `nuclear/` без изменения глобальной структуры.  
✅ **Чистота**: Корень `vault/` содержит только `_system/` (инфраструктура) и `nuclear/` (ресурсы).
