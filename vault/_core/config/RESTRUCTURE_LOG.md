# Лог реструктуризации `vault/` — v4 (Каноническая структура)

## Цель
Зафиксировать **каноническую структуру** `vault/` с строгим корнем, четким разделением глобального ядра и локальных контрактов, и агенто-дружелюбной навигацией.

## Изменения

### 1. Переименование глобальной инфраструктуры
- `_system/` → `_core/`
- **Почему:** `_core` явно указывает на "ядро системы", избегая путаницы с локальными системными папками ресурсов.

### 2. Переименование локального контракта ресурса
- `nuclear/engineering/_system/` → `nuclear/engineering/_contract/`
- **Почему:** `_contract` точно описывает содержимое (`_contract.yaml`, `dependencies.yaml`) и отличает его от глобального `_core/`.

### 3. Фиксация соседства `_contract/` и `_archive/`
- Локальный архив (`_archive/`) и локальный контракт (`_contract/`) теперь **ровные соседи** внутри ресурса.
- **Почему:** Логика (`_contract`) и данные (`_archive`) разделены. Это ускоряет чтение контракта ядром и упрощает семантику.

### 4. Строгий корень (Strict Root Policy)
Корень `vault/` теперь содержит **ТОЛЬКО**:
1. `_core/` — глобальная инфраструктура (обязательно)
2. `nuclear/` — контейнер ядерных ресурсов (обязательно)

**Запрещено:** любые другие файлы или папки на уровне корня `vault/`. Это правило обеспечивает предсказуемость для агентов и валидацию структуры.

## Итоговая каноническая структура

```
vault/
├── _core/                      # 🌍 ГЛОБАЛЬНОЕ ЯДРО
│   ├── config/                 # manifest.yaml, schema.yaml, security_policy.yaml
│   ├── context/                # принципы, процессы, память агентов, история
│   └── archive/                # глобальный архив всего проекта
│
└── nuclear/                    # 📦 КОНТЕЙНЕР РЕСУРСОВ
    └── engineering/            # Ресурс (можно плодить: legal/, research/, ...)
        ├── _contract/          # 📜 ПАСПОРТ РЕСУРСА
        │   ├── _contract.yaml  # типы, права, Зоны тьмы, внутренняя структура
        │   └── dependencies.yaml # зависимости от других ресурсов
        ├── _archive/           # 🗄 ИСТОРИЯ РЕСУРСА (сосед контракта)
        ├── workspace/          # ⚡ АКТИВНАЯ РАБОТА (опционально)
        └── artifacts/          # 🏛 ГОТОВАЯ ЦЕННОСТЬ (опционально)
```

## Преимущества канонической структуры
✅ **Агенто-дружелюбие**: Путь к контракту всегда один: `nuclear/{id}/_contract/_contract.yaml`. Zero-guessing.  
✅ **Чистота корня**: Строгое правило `[_core, nuclear]` исключает хаос.  
✅ **Семантическая ясность**: `_core` ≠ `_contract`. Глобальное ≠ локальное.  
✅ **Масштабируемость**: Новые ресурсы добавляются только в `nuclear/`, корень не пухнет.  
✅ **Мобильность**: Ресурс самодостаточен: скопировал `nuclear/engineering/` → получил контракт + историю + артефакты.  

## Сохранённость данных
✅ Все файлы сохранены, ни один артефакт не удалён.  
✅ Все `id` в фронтматтере сохранены (связи `[[id]]` не разорваны).
