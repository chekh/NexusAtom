# Vault как единый источник истины

## Тезис
`vault/` — централизованное хранилище всех знаний проекта. Код, контекст, правила, идеи, исследования живут в одном индексированном пространстве.

## Принцип
1. Физическая стабильность: файлы **никогда не перемещаются** при смене статуса. Меняется только `status` в метаданных.
2. ID-based навигация: все ссылки используют `[[id]]`, не относительные пути. Граф-резолвер подставляет актуальный путь и статус.
3. Разделение профилей: `delivery` (код/контракты) и `context` (заметки, ADR, исследования) используют один движок индексации, но разные правила валидации.

## Итог
Информация не расползается по папкам. Всё — узлы единого графа. Атом — лишь тяжёлый узел с жёстким контрактом. Остальное — контекстные узлы с гибким профилем.
