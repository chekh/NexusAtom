# Логическая архивация и целостность ссылок

## Тезис
Физическое перемещение в архив разрушает SSOT, ломает ссылки и требует ручного мерджа обобщающих документов. В NexusAtom архив — это **статус**, не папка.

## Принцип
1. `status: archived/deprecated` сохраняет файл на месте. Граф помечает узел неактивным.
2. `supersedes_by` / `supersedes` явно связывают старую и новую версии. История не теряется.
3. Динамическая агрегация: обзорные документы рендерятся из графа по `{{query: ...}}` шаблонам. Ручной мердж исключён.
4. Физическая очистка — опциональна (`nexus vault cleanup --physical`), требует валидации манифестов и явного подтверждения.

## Итог
Архивация не рвёт связи. Обобщающие документы всегда актуальны. Контекст сохраняется для аудита и онбординга.
